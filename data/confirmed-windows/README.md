# Confirmed-window exports

Git-tracked snapshots of harvest-window calibrations at the moment of
confirmation. Each file is a JSON dump generated by
`scripts/confirm-species.cjs` alongside the matching row in the
`confirmed_window_exports` table.

## Filename format

`<scientific-name-slugged>-<ISO-timestamp>.json`

Example: `acer-saccharum-2026-05-11T16-50-20-749Z.json`

Successive exports for the same species stack rather than overwrite —
the file is a receipt, never modified after creation. The most recent
timestamp per species is the active confirmed state.

## Restore from disk

If the DB is ever rolled back and a confirmed export is lost, the
windows + evidence can be replayed from the JSON dump. The file
contains every field needed to repopulate `species_fruiting_windows`
for that species at that snapshot.

## Workflow

```bash
# Confirm a species (snapshot + flip is_confirmed + dump JSON)
node scripts/confirm-species.cjs "Acer saccharum"

# With a human note about what changed since the last export
node scripts/confirm-species.cjs "Acer saccharum" \
  --notes "matches 2026 Cornell Maple Program bulletin v2"
```

After confirmation, the unify pipeline (`scripts/species-complex-unify.cjs`)
skips the species on future runs. To override (e.g. to re-incorporate
an improved unify entry), pass `--force-confirmed` to the unify
pipeline OR re-run `confirm-species.cjs` after editing the unify entry —
that creates a new export reflecting the current state.
